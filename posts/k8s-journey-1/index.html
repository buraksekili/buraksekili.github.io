Pods

Even though the pods might contain multiple containers, all are running on a single worker node; so, it doesn’t work on multiple worker nodes.

Resources

  1. Even you have multiple containers within a pod, each container shares the same IP address, port space, and Linux namespaces. So, be careful about port conflicts within a pod. But different pods do not have such a problem, since containers in a pod run in the same Network, not pods in a worker node.

  2. For local file system I/O, you probably do not have a common filesystem because most containers run upon different file systems. You can use volumes to share a common file system.
